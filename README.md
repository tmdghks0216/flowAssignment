# 파일 확장자 차단 과제
__요건__ <br/>
1-1. 고정 확장자는 차단을 자주하는 확장자를 리스트이며, default는 unCheck되어져 있습니다.<br/>
1-2. 고정 확장자를 check or uncheck를 할 경우 db에 저장됩니다. - 새로고침시 유지되어야합니다.<br/>
(아래쪽 커스텀 확장자에는 표현되지 않으니 유의해주세요.)<br/><br/>
2-1.  확장자 최대 입력 길이는 20자리<br/>
2-2. 추가버튼 클릭시 db 저장되며, 아래쪽 영역에 표현됩니다.<br/><br/>
3-1. 커스텀 확장자는 최대 200개까지 추가가 가능<br/>
3-2. 확장자 옆 X를 클릭시 db에서 삭제<br/><br/>

## 개발시 추가 고려사항 
__데이터 유효성__ <br/>
1-1. 대소문자 차이로 인한 중복 등록을 방지하기 위해 입력값을 소문자로 정규화하여 비교 및 저장하도록 개발 <br/>
1-2. 유효하지 않은 확장자 형식이 DB에 저장되는 것을 고려하여 한글 및 특수문자 입력 제한하도록 개발 <br/> <br/>
__2.사용자 가시성__ <br/>
2-1. 고정 확장자 체크/해제 시 매번 서버에 저장되므로 사용자에게 서버 반영 여부를 명확히 전달하기 위해 토스트 메세지 제공<br/><br/>
__3.응답 표준화__ <br/>
3-1. 신규 예외 케이스가 추가되더라도 확장성과 유지보수성을 고려하여 처리 결과를 표준화된 Enum 코드로 정의하여 컨트롤러에서 일관된 응답을 반환하도록 구성
